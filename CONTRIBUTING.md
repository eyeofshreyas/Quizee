# Contributing to Quizee

Thank you for considering contributing to Quizee!

We welcome bug fixes, improvements, documentation updates, and new features.

---

## Getting Started

1. Fork the repository.

2. Clone your fork.

```bash
git clone https://github.com/yourusername/quizee.git
```

3. Install dependencies.

```bash
npm install
```

4. Create a new branch.

```bash
git checkout -b feature/your-feature-name
```

---

## Development Guidelines

- Follow the existing project structure.
- Write clean, readable code.
- Keep commits small and meaningful.
- Test your changes before submitting.

---

## Commit Message Format

Examples:

```
feat: add leaderboard API

fix: resolve login validation issue

docs: update README

refactor: improve quiz scoring logic
```

---

## Pull Requests

Before submitting a Pull Request:

- Ensure the project runs correctly.
- Verify there are no unnecessary files.
- Update documentation if needed.
- Clearly describe your changes.

---

## Reporting Bugs

Please include:

- Description
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if applicable)

---

## Code Style

- Use meaningful variable names.
- Keep functions modular.
- Add comments where necessary.
- Follow JavaScript best practices.

---

Thank you for helping improve Quizee!
