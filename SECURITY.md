# Security Policy

## Supported Versions

Quizee is an educational and portfolio project. Security updates may be provided as the project evolves, but no long-term support is guaranteed.

---

## Reporting a Vulnerability

If you discover a security vulnerability, please avoid creating a public issue.

Instead, contact one of the project maintainers with:

- A description of the vulnerability
- Steps to reproduce it
- Any supporting screenshots or logs
- Suggested mitigation (if applicable)

The team will review the report and respond as soon as possible.

---

## Security Practices

Quizee follows several security best practices, including:

- JWT-based authentication
- Password hashing with bcrypt
- Input validation using Joi
- Environment variables for sensitive credentials
- Protected API routes

---

## Disclaimer

This project is intended for educational and portfolio purposes and should not be used in production environments without a full security review.
